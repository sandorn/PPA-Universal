# 贡献指南

感谢您对PPA项目的关注和支持！我们欢迎各种形式的贡献，无论是bug修复、新功能开发、文档改进还是其他类型的帮助。

## 行为准则

在参与本项目时，请遵守以下基本准则：

- 尊重所有贡献者
- 接受建设性批评
- 专注于最好地解决问题和改进项目

## 如何贡献

### 报告Bug

如果您发现了bug，请在GitHub上创建一个新的Issue，并包含以下信息：

1. 问题的简要描述
2. 重现问题的步骤
3. 预期行为
4. 实际行为
5. 您的环境信息（操作系统、PowerPoint版本等）
6. 可能的解决方案（如果您有）

### 提出新功能

如果您有新功能的想法，请创建一个Issue来讨论：

1. 功能描述
2. 为什么这个功能有用
3. 可能的实现方式

### 代码贡献

1. Fork项目仓库
2. 创建您的功能分支 (`git checkout -b feature/amazing-feature`)
3. 提交您的更改 (`git commit -m 'Add some amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 打开一个Pull Request

## 开发流程

1. 确保您的代码符合项目的代码风格
2. 为新功能添加适当的注释
3. 确保代码不会破坏现有功能
4. 在提交前测试您的更改

## 代码风格指南

- 遵循C#的标准编码规范
- 保持代码简洁、可读
- 使用有意义的变量和方法名
- 添加适当的注释说明复杂逻辑

## 联系方式

如有任何问题，可以通过GitHub Issues与项目维护者联系。

再次感谢您的贡献！
