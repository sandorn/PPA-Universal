# PPA-Universal 仓库目录说明

更新日期：[当前日期]

面向 PowerPoint/WPS 的分层 COM 加载项项目。本文只保留**目录职责与依赖**核心信息。

## 1) 架构总览

```
src/
├── PPA.Layered.sln
├── Core/
│   ├── PPA.Core/
│   └── PPA.Business/
├── Adapters/
│   ├── PPA.Adapter.PowerPoint/
│   └── PPA.Adapter.WPS/
└── Hosts/
    ├── PPA.Universal/
    └── PPA.Universal.ComAddIn/
tests/
└── PPA.Business.Tests/    # xUnit：对齐/分布等纯逻辑（`ShapeAlignmentMath`）
```

依赖方向：`Hosts -> Adapters + Core`，`Adapters -> Core`，`Business -> Core`，`Core` 不依赖上层。

## 2) 目录职责

| 路径 | 职责 |
| --- | --- |
| `src/Core/PPA.Core` | 抽象接口、配置、日志、DI 基础 |
| `src/Core/PPA.Business` | 平台无关业务服务与接口注册 |
| `src/Adapters/PPA.Adapter.PowerPoint` | PowerPoint（NetOffice）实现 |
| `src/Adapters/PPA.Adapter.WPS` | WPS（`dynamic` COM）实现 |
| `src/Hosts/PPA.Universal` | 平台检测、引导、`UniversalIntegration` |
| `src/Hosts/PPA.Universal.ComAddIn` | COM 入口、Ribbon XML、回调与图标资源 |

## 3) 两个 Host 的区别（重点）

| 路径 | 角色 | 说明 |
| --- | --- | --- |
| `src/Hosts/PPA.Universal` | 引擎层宿主库 | 负责平台识别与 DI 装配，对外暴露集成 API；不直接在 Office 勾选加载。 |
| `src/Hosts/PPA.Universal.ComAddIn` | Office COM 壳 | 实现 `IDTExtensibility2`，承载 Ribbon；通过 RegAsm 注册后在 Office/WPS 里启用。 |

简记：`PPA.Universal = 引擎`，`PPA.Universal.ComAddIn = 插件壳`。

## 4) 仓库根目录

| 路径 | 用途 |
| --- | --- |
| `README.md` | 面向使用者的快速说明 |
| `docs/README.md` | 面向开发者/贡献者的入口 |
| `build/` | 手动构建与注册脚本（如 `rebuild-register.bat`） |
| `ico/` | Ribbon 图标源（嵌入到 ComAddIn） |
| `docs/` | 设计与专题文档（含 `Ribbon-manual-regression.md`） |
| `tests/PPA.Business.Tests` | 单元测试（xUnit，`net48`，与 `PPA.Business` 同框架） |

## 5) 开发注意（超简）

- 新能力优先：`Business` 定义接口 -> 双 Adapter 实现 -> ComAddIn 接线。
- COM 对象注意释放；WPS/PPT 差异留在 Adapter。
- `bin/`、`obj/` 属构建产物，按源码目录理解结构。
